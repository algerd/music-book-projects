
Двухуровневая выборка данных:

1. Сырая или начальная:
    - для заполнения дерева и навигации по сущностям необходимы данные:
        - тип сущности
        - id сущности
        - name
        - parent
        - children
    Все эти данные прописаны в Entity.
    При первоначальной загрузке создаются коллекции объектов, соответствующие элементам таблиц,
    но заполненные не всеми данными, а только необходимыми для заполнения дерева и навигации.

2. Детальная:
    - при навигации по элементам для отображения всех данных необходимо подгружать данные 
    элемента дб. Для определения загружаемой сущности берётся сырой объект(его id).

3. Частичная:
    - в некоторых случаях нет надобности загружать все данные элемента дб, напр., в обзорных
    таблицах. И тогда, используя сырой объект(его id), подгружать дополнительные данные из бд.
